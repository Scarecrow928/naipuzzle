# Refactor: Scatter-and-Place Puzzle Mechanics

## Summary

Replace the sliding-puzzle mechanic with a scatter-and-place mechanic that includes a basket (暂存区). This significantly reduces difficulty. Default grid size returns to 4×4.

---

## Interaction Model (Click-to-Select → Click-to-Place)

**Canvas click when nothing is held:**
- Click **occupied tile** → Pick it up. Tile becomes empty. Piece becomes "held".
- Click **empty tile** → Nothing happens.

**Canvas click when a piece is held:**
- Click **empty tile** → Held piece placed there. Source position remains empty.
- Click **occupied tile** → Held piece placed there. Displaced piece goes into basket. Source remains empty.

**Basket interaction:**
- Basket is a bar at the bottom. Shows piece count. Click to expand/collapse.
- Expanded: scrollable horizontal list of piece thumbnails.
- Click a piece in the basket → That piece becomes "held". Basket closes.
- While a piece is held, clicking the basket area puts the held piece into the basket.
- Clicking the source tile (where piece was picked up from) puts it back (cancel).

**Cancel held piece:**
- Click the source tile again → piece returns to its original position.
- Click the basket → piece goes into the basket.

**Win condition:**
- All N×N pieces are in their correct grid positions (1→1, 2→2, ...). No pieces in basket. No pieces held.

---

## Data Model

```js
// Piece identity: 1..N*N (piece 1 belongs at grid position 0, piece 2 at position 1, etc.)

// Grid: array of length N*N, each element is pieceId (1-based) or null (empty)
grid = [3, null, 1, 2, 5, 6, null, 4, ...]

// Basket: array of pieceIds
basket = [7, 8]

// Held piece: the piece currently "in hand"
heldPiece = null | {
  pieceId: number,
  fromGridIndex: number | null,  // null if came from basket
}
```

**Initial state:**
- All grid positions filled (no empty spots)
- Pieces randomly shuffled across all positions
- Basket empty
- Held piece: null

## New `usePuzzle.js` API

```js
export function usePuzzle() {
  // State
  const gameState    // MENU | PLAYING | WIN
  const gridSize     // 4 (default)
  const grid         // array of pieceId|null (length N*N)
  const basket       // array of pieceId
  const heldPiece    // { pieceId, fromGridIndex } | null
  const imageUrl
  const moves
  const hasStarted
  const initialGridSnapshot   // for restart
  const initialBasketSnapshot // for restart

  // Actions
  startGame(size, imgUrl)
  restart()
  backToMenu()
  
  // Canvas interaction
  pickUpFromGrid(gridIndex)     // grid[pos] becomes null, heldPiece set
  placeOnGrid(gridIndex)        // place held piece at grid[pos], handle displaced
  cancelHold()                  // return held piece to source (grid or basket)
  
  // Basket interaction
  openBasket()                  // just a view state toggle
  pickUpFromBasket(pieceId)     // remove from basket, set as heldPiece
  dropIntoBasket()              // held piece → basket (if from grid)
  
  // Computed
  isSolved()                    // all pieces in correct position, no basket, no held
}
```

---

## Interaction Logic Detail

### `pickUpFromGrid(gridIndex)`:
- Precondition: `heldPiece === null`, grid[pos] has a piece
- Set `heldPiece = { pieceId: grid[pos], fromGridIndex: pos }`
- Set `grid[pos] = null`
- If first move, `hasStarted = true`

### `placeOnGrid(gridIndex)`:
- Precondition: `heldPiece !== null`
- If `grid[pos]` is null:
  - `grid[pos] = heldPiece.pieceId`
  - `heldPiece = null`
- If `grid[pos]` has a piece:
  - `basket.push(grid[pos])` (displaced piece goes to basket)
  - `grid[pos] = heldPiece.pieceId`
  - `heldPiece = null`
- Increment moves
- Check win: `isSolved()`

### `pickUpFromBasket(pieceId)`:
- Remove `pieceId` from basket
- Set `heldPiece = { pieceId, fromGridIndex: null }`

### `dropIntoBasket()`:
- Precondition: `heldPiece !== null`
- `basket.push(heldPiece.pieceId)`
- `heldPiece = null`

### `cancelHold()`:
- If `heldPiece.fromGridIndex !== null`: `grid[fromGridIndex] = heldPiece.pieceId`, `heldPiece = null`
- If `heldPiece.fromGridIndex === null`: `basket.push(heldPiece.pieceId)`, `heldPiece = null`

# Phase 2: Add Drag-and-Drop Support

## Summary

Add drag-and-drop alongside the existing click-to-place interaction. Both modes coexist — quick tap picks up/places, press-and-drag moves pieces between canvas cells and basket. Uses pointer events (works on desktop + mobile).

---

## Interaction Model (Updated)

### Click/Tap (preserved):
- Tap occupied tile → pick up. Tap any tile while holding → place.
- Tap basket piece → pick up from basket. Tap basket bar while holding → drop into basket.

### Drag (NEW):
- Press-and-hold on an occupied tile → pick up AND enter drag mode. A floating clone follows the pointer.
- Press-and-hold when a piece is already held → continue dragging to a new target.
- Release over a grid cell → place (empty=place, occupied=swap to basket)
- Release over the basket area → drop into basket
- Release over the source cell → cancel (return to original, no move counted)
- Release over invalid area → cancel (return to original, no move counted)

### Drag from basket (NEW):
- Press-and-hold on a piece in the expanded basket → pick up from basket AND enter drag mode.
- Release over grid cell → place (empty=place, occupied=swap to basket)
- Release over basket area → return to basket (cancel)
- Release over invalid area → return to basket (cancel)

### Visual feedback:
- Floating drag clone: piece thumbnail at pointer position, scale(1.05), shadow, z-index above everything
- Source cell: empty with dashed border + primary glow (existing)
- Hover target cell: highlighted border with primary color glow
- Hover target basket: basket bar border highlighted

---

## Pointer Events Design

Use pointer events exclusively for drag. Click/Tap for non-drag actions.

**Unified pointer flow:**

```
pointerdown on tile (occupied, nothing held):
  → pickUpFromGrid() → heldPiece set → dragArmed=true, record startPos

pointerdown on tile (occupied, piece held, tile != source):
  → dragArmed=true, record startPos on this target
  → (no pickup yet — piece already held)

pointerdown on tile (occupied, piece held, tile == source):
  → dragArmed=true, record startPos on source
  → (prepares for cancel)

pointermove (beyond 5px threshold):
  → isDragging = true
  → update dragPosition (floating clone position)
  → elementFromPoint() → update hoverTarget

pointerup:
  → if isDragging:
      if hoverTarget == grid cell → placeOnGrid(index)
      if hoverTarget == basket → dropIntoBasket()
      else → cancelHold()
  → else (was a tap):
      if heldPiece was already set before pointerdown → it was a placement tap → placeOnGrid(index)
      else → piece stays held (just picked up)
  → cleanup (remove listeners, reset flags)
```

**Key rules:**
- `pointerdown` always calls `event.preventDefault()` + `setPointerCapture()` to suppress native click and keep events on the source element
- Drag threshold: 5px movement before entering drag mode
- Click handler (native `@click`) is NOT used for drag path — pointer events handle everything

# Phase 3: Image-Aware Aspect Ratio + Custom Grid Dimensions

## Summary

Two tightly-related changes:
1. The puzzle board aspect ratio matches the source image (not forced 1:1), so cells may be non-square
2. Grid size selector removes 2×2/3×3, keeps 4×4/5×5 presets, adds a custom width×height input (2–10, configurable)

---

## Problem 1: Board Aspect Ratio

**Current**: Board always `aspect-ratio: 1`. All cells are square (`aspect-ratio: 1` on each tile).

**Target**: Board `aspect-ratio` = image natural ratio. Cells fill the grid proportionally — no individual cell `aspect-ratio` CSS rule needed, they fill `1fr` slots.

**Impact on data model**: `gridSize` (a single number) must become `gridColumns` + `gridRows`.

**Impact on rendering**:
- Board: `aspect-ratio: ${imageRatio}` (where imageRatio = width/height)
- Board max dimensions: must account for ratio. E.g. `max-width: min(90vw, calc((90vh - 160px) * imageRatio), 500px)`. If ratio < 1 (portrait), swap to constrain by height.
- Grid: `grid-template-columns: repeat(columns, 1fr)` + `grid-template-rows: repeat(rows, 1fr)`
- Tile background: `background-size: ${columns * 100}% ${rows * 100}%`; position calculated with separate col/row indexing
- Drag clone: size proportional to cell (width × height from board dimensions)

---

## Problem 2: Custom Grid Dimensions

**Current**: Size selector shows 2×2 through 6×6 as prefab buttons.

**Target**:
- Remove 2×2, 3×3, 6×6
- Keep 4×4 (default), 5×5 as preset buttons

- Validation: both ≥ 2, ≤ `MAX_GRID_SIZE` (from config)

## Aspect Ratio Layout Logic

```
Given: imageRatio = imgWidth / imgHeight
Board area constraints:
  maxW = min(90vw, 500px)
  maxH = calc(100vh - topBarHeight - basketHeight)  // ≈ calc(90vh - 160px)

If imageRatio >= 1 (landscape/square):
  boardWidth = min(maxW, maxH * imageRatio)
  boardHeight = boardWidth / imageRatio

If imageRatio < 1 (portrait):
  boardHeight = min(maxH, maxW / imageRatio)
  boardWidth = boardHeight * imageRatio
```

Applied as CSS:
```js
const boardStyle = {
  width: boardWidth + 'px',
  height: boardHeight + 'px',
  gridTemplateColumns: `repeat(${columns}, 1fr)`,
  gridTemplateRows: `repeat(${rows}, 1fr)`,
}
```

Instead of computing manually, use CSS clamping:
```css
.puzzle-board {
  aspect-ratio: var(--image-ratio);
  max-width: min(90vw, 500px);
  max-height: calc(90vh - 160px);
}
```

CSS `aspect-ratio` + `max-width` + `max-height` handles the fitting automatically — the browser picks the largest size that satisfies both aspect ratio and max constraints.

## Type-Safe Provide/Inject

Replace string-based inject with typed `InjectionKey`:

```
BEFORE:
  provide('theme', theme)
  const theme = inject('theme')

AFTER:
  provide(ThemeKey, theme)
  const theme = inject(ThemeKey)
```

Same for `drag` key. This avoids `any` types and gives full autocompletion.
